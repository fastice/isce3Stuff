## 1. Background and Motivation

Time-Domain Backprojection (TDBP) is one of the oldest and most geometrically exact SAR focusing algorithms. Although its computational cost — $\mathcal{O}(N^3)$ for a naïve implementation on an $N \times N$ scene — historically made it impractical at large scale, two developments have renewed its relevance:

1. **Fast Factorized Backprojection (FFBP)** reduces the complexity to $\mathcal{O}(N^2 \log N)$, comparable to frequency-domain methods.
2. **GPU acceleration (CUDA)** exploits the embarrassingly parallel structure of backprojection (each output pixel is independent) to achieve throughput that rivals or exceeds frequency-domain approaches on modern hardware.

ISCE3 adopts TDBP as the primary focusing engine for:

- **Airborne platforms** with non-linear trajectories (UAVSAR), where frequency-domain algorithms require complex motion compensation and introduce approximation errors.
- **Spotlight and circular-SAR modes**, where the non-rectangular data support makes frequency-domain methods difficult.
- **High-squint geometries** that break the assumptions of standard Range-Doppler or Chirp-Scaling algorithms.
- **Phase-preserving InSAR processing** where geometrical exactness is critical.

The algorithm is exposed in ISCE3 via `isce3.focus.backproject` (CPU) and `isce3.cuda.focus.backproject` (GPU).

## 2. SAR Signal Model

### 2.1 Transmitted Chirp

A SAR system transmits a linear frequency-modulated (LFM) pulse (chirp) at each pulse repetition interval. The complex baseband representation of the transmitted signal is:

$$s_T(\tau) = \text{rect}\!\left(\frac{\tau}{T_p}\right) \exp\!\left(j\pi K_r \tau^2\right)$$

where:

| Symbol | Meaning | Typical NISAR values |
|--------|---------|----------------------|
| $\tau$ | Fast time (range time), $\tau \in \left[-T_p/2,\, T_p/2\right]$ | — |
| $T_p$ | Pulse duration | ~20–80 µs |
| $K_r$ | Chirp rate (FM rate), $K_r = B/T_p$ | positive for up-chirp |
| $B$ | Range bandwidth | 20–80 MHz (NISAR) |

The instantaneous frequency of the chirp is:

$$f_i(\tau) = \frac{1}{2\pi}\frac{d}{d\tau}(\pi K_r \tau^2) = K_r \tau$$

The rect function is defined as:

$$\text{rect}(u) = \begin{cases} 1 & |u| \leq 1/2 \\ 0 & \text{otherwise} \end{cases}$$

### 2.2 Received Signal

For a point target at position $\mathbf{r}_t$ with radar cross-section $\sigma$, the received signal at slow time $\eta$ (azimuth time) is:

$$s_R(\tau, \eta) = \sigma \cdot A(\eta) \cdot s_T\!\left(\tau - \frac{2R(\eta)}{c}\right) \exp\!\left(-j\frac{4\pi f_c R(\eta)}{c}\right)$$

where:

| Symbol | Meaning |
|--------|---------|
| $\eta$ | Slow time (azimuth time) |
| $A(\eta)$ | Two-way antenna pattern weighting |
| $R(\eta)$ | Instantaneous slant range from platform to target |
| $c$ | Speed of light ($\approx 2.998 \times 10^8$ m/s) |
| $f_c$ | Radar carrier frequency |
| $\lambda = c/f_c$ | Radar wavelength |

The term $2R(\eta)/c$ is the two-way travel time; the exponential $\exp(-j4\pi f_c R/c)$ is the carrier phase — the quantity exploited in interferometry.

### 2.3 Demodulated (Baseband) Signal

After demodulation (mixing with the transmitted carrier), the received signal is expressed in terms of the **differential fast time** $\hat{\tau} = \tau - 2R(\eta)/c$:

$$s_d(\hat{\tau}, \eta) = A(\eta)\,\text{rect}\!\left(\frac{\hat{\tau}}{T_p}\right) \exp\!\left(j\pi K_r \hat{\tau}^2\right) \exp\!\left(-j\frac{4\pi R(\eta)}{\lambda}\right)$$

The first exponential carries the chirp phase within the pulse; the second carries the range-dependent carrier phase history used for azimuth compression.

### 2.4 Range History for a Point Target

Let $\mathbf{r}_s(\eta)$ be the platform position vector (ECEF) at slow time $\eta$, and $\mathbf{r}_t$ the target position. The instantaneous slant range is:

$$R(\eta) = \|\mathbf{r}_s(\eta) - \mathbf{r}_t\| = \sqrt{\bigl(x_s(\eta)-x_t\bigr)^2 + \bigl(y_s(\eta)-y_t\bigr)^2 + \bigl(z_s(\eta)-z_t\bigr)^2}$$

For a **zero-squint, linear-flight** geometry this simplifies to the classic hyperbolic approximation:

$$R(\eta) \approx \sqrt{R_0^2 + v^2\eta^2}$$

where $R_0$ is the closest-approach (zero-Doppler) range and $v$ is the effective platform velocity. ISCE3 does **not** use this approximation; it computes $R(\eta)$ exactly from the interpolated orbit at each slow time, which is why backprojection is superior to frequency-domain algorithms for non-linear trajectories.

The **Taylor expansion** around the zero-Doppler time $\eta_0$ is:

$$R(\eta) \approx R_0 + \dot{R}_0(\eta - \eta_0) + \frac{1}{2}\ddot{R}_0(\eta-\eta_0)^2 + \cdots$$

where $\dot{R}_0 = \partial R/\partial\eta|_{\eta=\eta_0}$ etc. The first-order term is the Doppler-rate-induced phase ramp; the second-order term creates the Doppler chirp (azimuth FM rate).

## 3. The Backprojection Integral

### 3.1 Continuous Formulation

After range compression (Section 4), the range-compressed signal $s_{rc}(\tau, \eta)$ is a compressed pulse in the $\tau$ direction. Backprojection forms the focused complex image $I(\mathbf{r}_t)$ by integrating (coherently summing) contributions from all slow-time positions across the synthetic aperture $[\eta_1, \eta_2]$:

$$\boxed{I(\mathbf{r}_t) = \int_{\eta_1}^{\eta_2} s_{rc}\!\left(\frac{2R(\eta;\mathbf{r}_t)}{c},\, \eta\right) \exp\!\left(+j\frac{4\pi R(\eta;\mathbf{r}_t)}{\lambda}\right) A^*(\eta)\, d\eta}$$

This is the **fundamental backprojection equation**. Each term has a clear physical interpretation:

- $s_{rc}(2R/c,\, \eta)$: Extracts the sample from the range-compressed data at the two-way travel time for target $\mathbf{r}_t$ at slow time $\eta$.
- $\exp(+j4\pi R/\lambda)$: **Phase deramping** — removes the carrier phase history accumulated during imaging, phase-aligning all aperture contributions so they add coherently.
- $A^*(\eta)$: Optional matched weighting for antenna pattern ($A^*$ is the conjugate of the illumination pattern).

### 3.2 Discrete Implementation

In practice, slow time is sampled at the PRF: $\eta_n = \eta_1 + n/\text{PRF}$, $n = 0, 1, \ldots, N_a - 1$. The discrete backprojection sum is:

$$I(\mathbf{r}_t) = \sum_{n=0}^{N_a - 1} w_n \cdot \mathcal{I}\!\left\lbrace s_{rc}(\cdot,\, n),\, \frac{2R(\eta_n;\mathbf{r}_t)}{c}\right\rbrace \exp\!\left(+j\frac{4\pi R(\eta_n;\mathbf{r}_t)}{\lambda}\right)$$

where:

- $\mathcal{I}\{s_{rc}(\cdot,n),\, \tau_0\}$ denotes **range interpolation** of the $n$-th range line at the fractional sample position $\tau_0$ (see Section 7).
- $w_n$ is a taper/window weight (e.g., Taylor, Hamming) applied at slow time $n$ to control azimuth sidelobes.
- $N_a$ is the number of pulses in the synthetic aperture.

### 3.3 Phase Correction Term

The phase correction $\phi_{\text{BP}}(\eta_n, \mathbf{r}_t)$ applied at each aperture position is:

$$\phi_{\text{BP}}(\eta_n, \mathbf{r}_t) = \frac{4\pi}{\lambda} R(\eta_n;\mathbf{r}_t) = \frac{4\pi f_c}{c} \|\mathbf{r}_s(\eta_n) - \mathbf{r}_t\|$$

This is the **two-way phase**, i.e., twice the one-way phase (since both outgoing and return paths accumulate phase). The sign convention (positive in ISCE3's deramping convention) means the phase correction undoes the $-4\pi R/\lambda$ accumulated in the received signal.

For a stationary point target, after summing over all $N_a$ aperture positions, all contributions add constructively and the total focused amplitude is proportional to $N_a$.

**Residual carrier phase.** In the focused image, the absolute phase of pixel $I(\mathbf{r}_t)$ is:

$$\angle I(\mathbf{r}_t) = -\frac{4\pi R_0}{\lambda} + \phi_{\text{ref}} + \phi_{\text{atm}} + \phi_{\text{noise}}$$

where $R_0$ is the zero-Doppler slant range, $\phi_{\text{ref}}$ is a reference phase (absorbed into the image), and $\phi_{\text{atm}}$ is the atmospheric delay. This **preserved phase** is what makes backprojection suitable for InSAR.

## 4. Range Compression

Range compression converts each transmitted pulse from a time-bandwidth product $>1$ chirp into a compressed pulse approaching the diffraction limit in the range direction. This is done prior to backprojection.

### 4.1 Matched Filter

The matched filter (MF) for the transmitted chirp is the time-reversed, complex-conjugated replica:

$$h_{\text{MF}}(\tau) = s_T^*(-\tau) = \text{rect}\!\left(\frac{\tau}{T_p}\right)\exp\!\left(-j\pi K_r \tau^2\right)$$

The range-compressed signal is the convolution:

$$s_{rc}(\tau, \eta) = s_d(\tau, \eta) * h_{\text{MF}}(\tau)$$

### 4.2 Pulse Compression in Frequency Domain

Because convolution in time equals multiplication in frequency, range compression is efficiently computed via FFT:

$$S_{rc}(f_\tau, \eta) = S_d(f_\tau, \eta) \cdot H_{\text{MF}}(f_\tau)$$

where $H_{\text{MF}}(f_\tau) = T_p\,\text{rect}(f_\tau / B)\exp(+j\pi f_\tau^2 / K_r)$ is the matched filter transfer function.

The compressed pulse envelope in fast time is approximately a sinc function:

$$s_{rc}(\tau, \eta) \approx T_p B \cdot \text{sinc}\!\left(B\tau\right) \exp\!\left(-j\frac{4\pi f_c R(\eta)}{\lambda}\right)$$

The amplitude is $T_p B$ (the time-bandwidth product, i.e., the pulse compression gain), and the phase retains the full carrier history needed for backprojection.

### 4.3 Range Resolution

The slant-range resolution after pulse compression is:

$$\delta_r = \frac{c}{2B}$$

The corresponding **ground-range resolution** depends on the local incidence angle $\theta_i$:

$$\delta_{gr} = \frac{c}{2B\sin\theta_i}$$

| Bandwidth $B$ | Slant-range resolution $\delta_r$ |
|---|---|
| 20 MHz | 7.5 m |
| 40 MHz | 3.75 m |
| 80 MHz | 1.875 m |
| 100 MHz | 1.5 m |

## 5. Azimuth Processing and Synthetic Aperture

### 5.1 Synthetic Aperture Length

The synthetic aperture length (in slow time) over which a target at range $R_0$ is illuminated is:

$$T_{SA} = \frac{\lambda R_0}{v_g \cdot D_a}$$

where $v_g$ is the platform ground velocity, $D_a$ is the real antenna length in the azimuth dimension, and $\lambda$ is the wavelength. The physical aperture length in space is $L_{SA} = v \cdot T_{SA}$.

### 5.2 Azimuth Resolution

The one-look azimuth (along-track) resolution for a focused SAR image is:

$$\delta_a = \frac{D_a}{2}$$

This remarkable result — independent of range and wavelength — arises because the synthetic aperture length scales with range and wavelength in exactly the right way to keep resolution constant. The factor of $1/2$ comes from the two-way (coherent two-pass) effect.

The **Doppler bandwidth** processed determines azimuth resolution:

$$\delta_a = \frac{v_g}{B_D}$$

where $B_D$ is the processed Doppler bandwidth.

### 5.3 Doppler Centroid and Bandwidth

For a platform velocity $\mathbf{v}_s$ and a target-to-sensor look vector $\hat{\mathbf{r}}$, the Doppler centroid frequency is:

$$f_{DC} = -\frac{2}{\lambda}\, \mathbf{v}_s \cdot \hat{\mathbf{r}}_{||} = -\frac{2 v_s \sin\theta_{\text{sq}}}{\lambda}$$

where $\theta_{\text{sq}}$ is the squint angle and $\hat{\mathbf{r}}_{||}$ is the component of $\hat{\mathbf{r}}$ along the flight path.

The **Doppler chirp rate** (azimuth FM rate) is:

$$K_a = -\frac{2 v_s^2 \cos^2\!\theta_{\text{sq}}}{\lambda R_0}$$

The Doppler bandwidth illuminated by the antenna beam is:

$$B_D \approx \frac{0.886\, v_s \cos\theta_{\text{sq}}}{D_a / 2} = \frac{0.886 \cdot 2 v_s \cos\theta_{\text{sq}}}{D_a}$$

In backprojection, these quantities enter implicitly through the exact range history $R(\eta; \mathbf{r}_t)$; there is no need to estimate or compensate $f_{DC}$ explicitly.

## 6. Geometry in ISCE3

### 6.1 Coordinate Systems

ISCE3 works in **Earth-Centered Earth-Fixed (ECEF)** Cartesian coordinates $(X, Y, Z)$ internally. All orbit state vectors, target positions, and range computations are performed in ECEF. Geodetic coordinates (latitude $\phi$, longitude $\lambda$, ellipsoidal height $h$) are converted to/from ECEF via the WGS84 ellipsoid transformation:

$$\begin{pmatrix}X\\Y\\Z\end{pmatrix} = \begin{pmatrix}(N+h)\cos\phi\cos\lambda\\ (N+h)\cos\phi\sin\lambda\\ (N(1-e^2)+h)\sin\phi\end{pmatrix}$$

where $N = a/\sqrt{1-e^2\sin^2\phi}$ is the prime vertical radius of curvature, $a = 6{,}378{,}137$ m is the WGS84 semi-major axis, and $e^2 = 0.00669437999014$ is the first eccentricity squared.

### 6.2 Platform Position and Velocity

At each slow time $\eta_n$, ISCE3 computes the **platform state vector**:

$$\mathbf{s}(\eta_n) = \begin{pmatrix}\mathbf{r}_s(\eta_n)\\ \mathbf{v}_s(\eta_n)\end{pmatrix} \in \mathbb{R}^6$$

where $\mathbf{r}_s \in \mathbb{R}^3$ (ECEF position, meters) and $\mathbf{v}_s \in \mathbb{R}^3$ (ECEF velocity, m/s).

The **slant range** to a target $\mathbf{r}_t$ is then:

$$R(\eta_n;\mathbf{r}_t) = \|\mathbf{r}_s(\eta_n) - \mathbf{r}_t\|_2$$

The **unit look vector** from the platform to the target is:

$$\hat{\mathbf{u}}(\eta_n) = \frac{\mathbf{r}_t - \mathbf{r}_s(\eta_n)}{R(\eta_n;\mathbf{r}_t)}$$

### 6.3 Orbit Interpolation (Hermite)

ISCE3's orbit module uses **Hermite cubic interpolation** between adjacent state vectors. Given state vectors at times $\eta_k$ and $\eta_{k+1}$ with positions $\mathbf{r}_k$, $\mathbf{r}_{k+1}$ and velocities $\mathbf{v}_k$, $\mathbf{v}_{k+1}$, the interpolated position at $\eta \in [\eta_k, \eta_{k+1}]$ is:

$$\mathbf{r}_s(\eta) = h_{00}(t)\,\mathbf{r}_k + h_{10}(t)\Delta\eta\,\mathbf{v}_k + h_{01}(t)\,\mathbf{r}_{k+1} + h_{11}(t)\Delta\eta\,\mathbf{v}_{k+1}$$

where $t = (\eta - \eta_k)/\Delta\eta$, $\Delta\eta = \eta_{k+1} - \eta_k$, and the **Hermite basis functions** are:

$$\begin{aligned}
h_{00}(t) &= 2t^3 - 3t^2 + 1\\
h_{10}(t) &= t^3 - 2t^2 + t\\
h_{01}(t) &= -2t^3 + 3t^2\\
h_{11}(t) &= t^3 - t^2
\end{aligned}$$

This guarantees continuity of both position and velocity at each state-vector epoch, yielding geolocation accuracy on the order of a few millimetres for state vectors separated by $\leq 30$ seconds.

The interpolated velocity is obtained by differentiating the Hermite polynomial:

$$\mathbf{v}_s(\eta) = \frac{d\mathbf{r}_s}{d\eta} = \frac{1}{\Delta\eta}\!\left[\dot{h}_{00}(t)\,\mathbf{r}_k + \dot{h}_{10}(t)\Delta\eta\,\mathbf{v}_k + \dot{h}_{01}(t)\,\mathbf{r}_{k+1} + \dot{h}_{11}(t)\Delta\eta\,\mathbf{v}_{k+1}\right]$$

### 6.4 DEM Integration

Backprojection requires knowing the **target height** $h_t$ to resolve the range-azimuth-height ambiguity. ISCE3 uses an external DEM raster (see Section 3 of the user manual). At each output pixel $(\phi_t, \lambda_t)$, the height $h_t = \text{DEM}(\phi_t, \lambda_t)$ is bilinearly interpolated from the DEM, then converted to ECEF via the WGS84 equations above to obtain $\mathbf{r}_t$.

> **Important:** The DEM must be referenced to the WGS84 **ellipsoid**, not a geoid (e.g., EGM96). An error of $\Delta h$ in the DEM introduces a geolocation error $\Delta h / \tan\theta_i$ in ground range, where $\theta_i$ is the local incidence angle.

### 6.5 Squint Angle

The squint angle $\theta_{\text{sq}}$ is the angle between the zero-Doppler direction (broadside) and the actual look direction in the along-track plane:

$$\sin\theta_{\text{sq}} = \frac{-\lambda f_{DC}}{2\|\mathbf{v}_s\|}$$

For **zero-squint (broadside)** imaging, $f_{DC} = 0$ and the Doppler centroid LUT contains zeros — corresponding to `isce3.core.LUT2d()` with all-zero values in the backprojection call.

For **squinted** acquisitions, the Doppler centroid LUT must be populated with the estimated centroid values so that the correct integration limits in slow time can be determined and the correct aperture segment is summed for each pixel.

## 7. Interpolation Kernels

After range compression, the range-compressed data is sampled at the discrete fast-time grid $\tau_m = \tau_0 + m/f_s$. Backprojection requires evaluating $s_{rc}$ at the **exact** two-way travel time $2R(\eta_n;\mathbf{r}_t)/c$, which is generally not aligned with the sample grid. A range interpolation step is therefore required for every (aperture position, output pixel) pair.

The ideal interpolator is the band-limited sinc function, but a truncated and windowed approximation is used in practice.

### 7.1 Sinc Interpolation

The ideal reconstruction formula:

$$s_{rc}(\tau_0) = \sum_{m=-\infty}^{\infty} s_{rc}(\tau_m) \cdot \text{sinc}\!\left(\frac{\tau_0 - \tau_m}{1/f_s}\right)$$

In ISCE3, a truncated sinc kernel of length $K$ (typically $K = 8$ or $K = 9$ taps) is used:

$$s_{rc}(\tau_0) \approx \sum_{m=m_0}^{m_0+K-1} s_{rc}(\tau_m) \cdot \text{sinc}\!\left(\frac{\tau_0 - \tau_m}{1/f_s}\right)$$

Peak sidelobe ratio (PSLR) approaches $-13.3$ dB for a rectangular-windowed sinc.

### 7.2 Lanczos (Windowed Sinc)

The Lanczos kernel of order $a$ applies a sinc window to reduce the Gibbs phenomenon:

$$L_a(x) = \begin{cases} \text{sinc}(x)\cdot\text{sinc}(x/a) & |x| < a \\ 0 & |x| \geq a \end{cases}$$

This is equivalent to convolving the ideal sinc with a second, wider sinc — suppressing aliasing at the cost of a small resolution loss. ISCE3 uses $a = 4$ (8-tap kernel) as a default, offering the best trade-off between aliasing suppression, sharpness, and ringing.

### 7.3 Cubic Convolution

A piecewise cubic approximation to the sinc:

$$h(x) = \begin{cases} \tfrac{3}{2}|x|^3 - \tfrac{5}{2}|x|^2 + 1 & |x| \leq 1 \\ -\tfrac{1}{2}|x|^3 + \tfrac{5}{2}|x|^2 - 4|x| + 2 & 1 < |x| \leq 2 \\ 0 & |x| > 2 \end{cases}$$

This 4-tap kernel is faster to evaluate than the sinc but has somewhat higher sidelobes (PSLR $\approx -24$ dB).

### 7.4 Phase-Controlled Interpolation for InSAR

Standard amplitude interpolation methods applied to complex SLC data can introduce phase errors when the interferometric phase varies rapidly across samples. ISCE3 supports **phase-controlled interpolation** in which the interpolation is performed after modulating each sample to remove the expected phase ramp across the kernel:

$$I_{\text{interp}} = \exp\!\left(j\phi_{\text{ref}}\right) \cdot \sum_m w_m \cdot s_{rc}(\tau_m) \exp\!\left(-j\phi(\tau_m)\right)$$

where $\phi(\tau_m) = 4\pi f_c \tau_m / 2$ is a linearised phase model and $\phi_{\text{ref}}$ is the phase evaluated at the interpolation point $\tau_0$. This approach is recommended for InSAR applications where phase accuracy is paramount.

## 8. Computational Complexity

### 8.1 Naive Backprojection — $\mathcal{O}(N^3)$

For a scene of $N \times N$ output pixels and $N_a = N$ aperture pulses, the naive backprojection requires:

- **Range interpolation:** 1 per (pixel, pulse) pair $\Rightarrow$ $N^2 \times N = N^3$ operations
- **Phase rotation:** $e^{j\phi}$ per (pixel, pulse) pair $\Rightarrow$ $N^3$ operations
- **Accumulation:** $N^3$ complex additions

Total: $\mathcal{O}(N^3)$ — prohibitively expensive for large scenes without a GPU.

### 8.2 Fast Factorized Backprojection — $\mathcal{O}(N^2 \log N)$

The **Fast Factorized Backprojection (FFBP)** algorithm (Ulander et al. 2003; Duersch 2013) exploits the fact that contributions from a long aperture can be recursively fused from two shorter sub-apertures:

$$I_{L}(\mathbf{r}_t) = I_{L/2,\text{left}}(\mathbf{r}_t) \cdot \exp(j\phi_{\text{left}}) + I_{L/2,\text{right}}(\mathbf{r}_t) \cdot \exp(j\phi_{\text{right}})$$

This **divide-and-conquer** decomposition has $\log_2 N$ levels, each requiring $\mathcal{O}(N^2)$ operations, yielding:

$$\text{FFBP complexity:}\quad \mathcal{O}(N^2 \log_2 N)$$

The reduction factor vs naive backprojection for a 10,000-pulse aperture is approximately $N / \log_2 N \approx 750\times$.

### 8.3 Comparison with Frequency-Domain Algorithms

| Algorithm | Complexity | Phase accuracy | Non-linear motion | Large squint | DEM support |
|---|---|---|---|---|---|
| Naive backprojection | $\mathcal{O}(N^3)$ | Exact | Native | Native | Native |
| FFBP | $\mathcal{O}(N^2\log N)$ | Exact | Native | Native | Native |
| Range-Doppler (RDA) | $\mathcal{O}(N^2\log N)$ | Good | Requires MoCo | Limited | Post-focus |
| Chirp Scaling (CSA) | $\mathcal{O}(N^2\log N)$ | Very good | Requires MoCo | Limited | Post-focus |
| Omega-K (WDA) | $\mathcal{O}(N^2\log N)$ | Excellent | Requires MoCo | Moderate | Post-focus |
| Polar Format (PFA) | $\mathcal{O}(N^2\log N)$ | Good | Requires MoCo | Native | Post-focus |

**Key differentiators of backprojection:**
- No approximations to the range history equation — exact $R(\eta;\mathbf{r}_t)$ used for every (pixel, pulse)
- No Range Cell Migration Correction (RCMC) step — it is implicit in the range interpolation
- No bulk or differential motion compensation step — the exact orbit naturally handles any trajectory
- DEM-driven: height information is directly embedded in the focussed-image pixel definition

## 9. GPU (CUDA) Implementation

### 9.1 Parallelisation Strategy

Backprojection is **embarrassingly parallel** at the pixel level: the contribution of each aperture position to each output pixel is computed independently. ISCE3's CUDA implementation maps this structure directly to the GPU:

```
Grid layout:
  Grid  → tiles of output image blocks
  Block → threads, each responsible for one (or a strip of) output pixel(s)

For each CUDA thread computing pixel (i, j):
  accum = 0 + 0j
  for n in range(N_aperture):
      r_t   = ecef(dem[i,j], phi[i], lam[j])  # from DEM + lat/lon grid
      r_s_n = orbit.interpolate(eta[n])         # platform position
      R_n   = norm(r_s_n - r_t)                 # slant range
      tau_n = 2 * R_n / c                        # two-way travel time
      s     = range_interp(s_rc[:, n], tau_n)   # interpolate range line
      phi_n = 4 * pi * R_n / wavelength          # phase correction
      accum += w[n] * s * exp(1j * phi_n)
  image[i, j] = accum
```

The inner loop over aperture positions is sequential within a thread (this loop cannot be trivially parallelised because of the accumulation).

### 9.2 Memory Hierarchy Optimisation

The dominant cost in the GPU implementation is **memory bandwidth** — loading the range-compressed data $s_{rc}$ from global memory for each (pixel, pulse) pair. ISCE3 applies three tiers of optimisation:

**Tiled data loading (shared memory)**

The range-compressed data for a block of aperture positions is cooperatively loaded by all threads in a CUDA block into **shared memory** before the inner loop. Since adjacent pixels tend to need nearby range samples, this dramatically improves cache hit rates.

**Constant memory for orbit and LUTs**

The orbit state vectors, Doppler LUT, and window coefficients are stored in GPU **constant memory** (64 KB per SM, broadcast to all threads in a warp) since they are read-only and accessed uniformly across all threads.

**Texture memory for range interpolation**

The range-compressed data (or the interpolation kernel itself) can be placed in **texture memory** to exploit the hardware's built-in 2D spatial caching and linear interpolation (for bilinear texture interpolation as a fast approximation).

**Memory access pattern**

Threads within a CUDA warp (32 threads) are assigned to **adjacent output pixels** in the range direction. This ensures that range interpolation requests for the same aperture position but adjacent pixels map to nearby range samples — enabling coalesced global memory reads.

### 9.3 Performance Characteristics

Benchmarks reported for ISCE3-based CUDA backprojection:

| Platform | Relative speed |
|---|---|
| Single-thread CPU (MATLAB) | 1× (baseline) |
| Multi-thread CPU (OpenMP, 16 cores) | ~8–15× |
| NVIDIA GPU (CUDA) | ~200× |

On a modern NVIDIA A100 GPU with 80 GB HBM2e, ISCE3's CUDA backprojection can focus a full NISAR stripmap frame ($\sim10{,}000$ range lines × 50,000 range samples, 10,000-pulse aperture) in a few minutes, compared to hours on a multi-core CPU.

The GPU speedup scales roughly as:

$$S_{\text{GPU}} \approx \frac{N_{\text{cores,GPU}} \cdot f_{\text{mem,GPU}}}{N_{\text{cores,CPU}} \cdot f_{\text{mem,CPU}}} \cdot \eta_{\text{eff}}$$

where $\eta_{\text{eff}}$ is the parallelisation efficiency (typically 0.6–0.9 for backprojection, limited by memory bandwidth).

## 10. Sub-Aperture and Fast Backprojection

### Sub-Aperture Decomposition

The full synthetic aperture of length $L$ is partitioned into $M$ sub-apertures of length $L/M$:

$$I_L(\mathbf{r}_t) = \sum_{m=0}^{M-1} I_{L/M}^{(m)}(\mathbf{r}_t)$$

Each sub-aperture image $I_{L/M}^{(m)}$ is formed independently (embarrassingly parallel over sub-apertures), then the sub-aperture images are coherently summed. This enables:

- **Memory management:** Only the data for one sub-aperture needs to be resident on the GPU at a time.
- **Multi-GPU processing:** Different sub-apertures assigned to different GPUs, results merged on the host.
- **Sub-aperture InSAR:** Comparing sub-aperture images from different azimuth looks enables along-track interferometry (ATI) for measuring ocean currents or target velocity.

### Fast Factorized Backprojection (FFBP)

FFBP uses recursive polar-to-Cartesian sub-image merging. At level $k$ of the recursion, $2^k$ sub-aperture images are available in local polar coordinates centred on the mid-aperture platform position. Two adjacent sub-images (in polar coordinates) are merged by:

1. Resampling both sub-images onto a common local polar grid.
2. Phase-correcting each sub-image for the offset between its local reference point and the merged image's reference point.
3. Coherently summing the phase-corrected sub-images.

The merge phase correction for sub-aperture $m$ when forming the merged image at reference position $\mathbf{r}_{\text{ref}}^{(\text{merged})}$ is:

$$\Delta\phi^{(m)} = \frac{4\pi}{\lambda}\!\left(R(\mathbf{r}_{\text{ref}}^{(m)}; \mathbf{r}_t) - R(\mathbf{r}_{\text{ref}}^{(\text{merged})}; \mathbf{r}_t)\right)$$

This correction is small (varies slowly with $\mathbf{r}_t$) and can be computed on a coarsened grid then interpolated, giving the $\mathcal{O}(N^2\log N)$ speedup.

## 11. Phase Preservation and InSAR Considerations

### Phase Budget

For a single-pass InSAR pair (reference $R$ and secondary $S$, both focused with backprojection to the same DEM-driven pixel grid), the interferometric phase at pixel $\mathbf{r}_t$ is:

$$\Delta\phi(\mathbf{r}_t) = \angle\!\left[I_R(\mathbf{r}_t)\cdot I_S^*(\mathbf{r}_t)\right]$$

Expanding using the backprojection phase:

$$\Delta\phi = -\frac{4\pi}{\lambda}\left(R_0^R - R_0^S\right) + \Delta\phi_{\text{topo}} + \Delta\phi_{\text{defo}} + \Delta\phi_{\text{atm}} + \Delta\phi_{\text{noise}}$$

where:

| Term | Expression | Meaning |
|---|---|---|
| $-\frac{4\pi}{\lambda}(R_0^R - R_0^S)$ | Flat-earth phase | Carrier phase difference at reference ellipsoid |
| $\Delta\phi_{\text{topo}}$ | $\frac{4\pi B_\perp h}{\lambda R_0 \sin\theta_i}$ | Topographic phase (proportional to perpendicular baseline $B_\perp$) |
| $\Delta\phi_{\text{defo}}$ | $\frac{4\pi \Delta r_{\text{LOS}}}{\lambda}$ | Surface deformation in line-of-sight |
| $\Delta\phi_{\text{atm}}$ | — | Differential ionospheric + tropospheric delay |
| $\Delta\phi_{\text{noise}}$ | — | Thermal noise, temporal decorrelation |

**Backprojection advantage:** Because the exact $R(\eta_n; \mathbf{r}_t)$ is used (not an approximation), the flat-earth and topographic components are correctly computed for any baseline geometry without additional baseline-dependent phase corrections.

### Coherence

The interferometric coherence is:

$$\gamma = \frac{|\langle I_R\, I_S^*\rangle|}{\sqrt{\langle|I_R|^2\rangle\langle|I_S|^2\rangle}}$$

Coherence is bounded by $[0, 1]$ and is maximised when the two images are focused to the same exact geometry. Since backprojection uses the same DEM and the same interpolation kernel for both reference and secondary, any coherence loss due to focusing-geometry mismatch is minimised.

### Common-Band Filtering

For InSAR formed from two acquisitions with different Doppler centroids $f_{DC}^R \neq f_{DC}^S$, only the common azimuth spectral overlap contributes coherent phase. ISCE3 applies **common-band Doppler filtering** before or during backprojection to restrict each SLC to the shared bandwidth, improving coherence.

Similarly, if the perpendicular baseline $B_\perp$ is non-zero, the two acquisitions view the ground from slightly different angles, creating a frequency shift in the range spectrum. **Common-band range filtering** retains only the overlap, removing the baseline decorrelation contribution.

### 12.3 Worked Example

The following end-to-end example follows the pattern used in ISCE3's own test suite (`tests/python/extensions/pybind/focus/backproject.py`). It focuses a point-target simulation stored in HDF5:

```python
import isce3
import isce3.ext.isce3 as isce
import h5py
import numpy as np
from isce3.core import load_orbit_from_h5_group

c = isce.core.speed_of_light

# ── 1. Load range-compressed data from HDF5 ──────────────────────────────────
with h5py.File("point-target-sim-rc.h5", "r") as f:
    signal_data      = f["data"][()]                          # complex64 array
    orbit            = load_orbit_from_h5_group(f["orbit"])
    doppler          = isce.core.LUT2d.load_from_h5(f["doppler"], "doppler")
    sensing_start    = f["time_of_first_pulse"][()]
    az_spacing       = f["pulse_spacing"][()]
    two_way_delay    = f["two_way_range_delay"][()]
    rg_sample_rate   = f["range_sample_rate"][()]
    center_freq      = f["center_frequency"][()]
    look_side        = f["look_side"][()]
    terrain_height   = f["terrain_height"][()]
    apply_atm        = f["atmosphere_model"][()]

lines, samples = signal_data.shape
near_range  = c / 2. * two_way_delay
rg_spacing  = c / (2. * rg_sample_rate)
wavelength  = c / center_freq
prf         = 1. / az_spacing

# ── 2. Build input RadarGeometry (grid + orbit + Doppler) ────────────────────
in_grid = isce.product.RadarGridParameters(
    sensing_start, wavelength, prf, near_range, rg_spacing,
    look_side, lines, samples, orbit.reference_epoch)
in_geometry = isce.container.RadarGeometry(in_grid, orbit, doppler)

# ── 3. Build output RadarGeometry (a small chip centred on the target) ────────
target_azimuth = ...   # seconds from epoch (loaded from HDF5)
target_range   = ...   # metres
nchip = 129
dt = in_grid.az_time_interval
dr = in_grid.range_pixel_spacing
t0 = target_azimuth - 0.5 * (nchip - 1) * dt
r0 = target_range   - 0.5 * (nchip - 1) * dr
out_grid = isce.product.RadarGridParameters(
    t0, wavelength, prf, r0, dr, look_side, nchip, nchip, orbit.reference_epoch)
out_geometry = isce.container.RadarGeometry(out_grid, orbit, doppler)

# ── 4. DEM and interpolation kernel ──────────────────────────────────────────
dem = isce.geometry.DEMInterpolator(terrain_height)   # constant-height DEM

B = 20e6          # range bandwidth (Hz)
azimuth_res = 6.  # desired azimuth resolution (m)

# Knab kernel, 9 samples wide; tabulate for performance
kernel = isce.core.KnabKernel(9., B / rg_sample_rate)
kernel = isce.core.TabulatedKernelF32(kernel, 2048)

# ── 5. Allocate output buffers ────────────────────────────────────────────────
out    = np.empty((nchip, nchip), np.complex64)
height = np.empty((nchip, nchip), np.float32)

dry_tropo_model = "tsx" if apply_atm else "nodelay"

# ── 6. Run backprojection (CPU) ───────────────────────────────────────────────
err = isce.focus.backproject(
    out, out_geometry,
    signal_data, in_geometry,
    dem, center_freq, azimuth_res, kernel,
    dry_tropo_model,
    height=height
)
assert not err, f"Backprojection geometry failed to converge: {err}"

print(f"Focusing complete. Output shape: {out.shape}")
print(f"Pixel heights (m above ellipsoid): {height.mean():.1f}")

# ── 7. (Optional) GPU version — same positional args, adds batch ──────────────
if hasattr(isce, "cuda"):
    out_gpu = np.empty_like(out)
    err = isce.cuda.focus.backproject(
        out_gpu, out_geometry,
        signal_data, in_geometry,
        dem, center_freq, azimuth_res, kernel,
        dry_tropo_model,
        batch=1024,
        height=height
    )
    assert not err
```

## 14. Comparison: Backprojection vs Other ISCE3 Focusers

ISCE3 supports multiple SAR focusing algorithms. The table below guides algorithm selection:

| Criterion | Backprojection (TDBP) | Range-Doppler (RDA) | Chirp Scaling (CSA) |
|---|---|---|---|
| **Geometric accuracy** | Exact (no approximations) | Good (Doppler approx.) | Very good |
| **Non-linear trajectory** | Native — no MoCo needed | Requires MoCo pre-processing | Requires MoCo |
| **Large squint** | Native | Degraded > ~10° | Degraded |
| **Wide swath** | Native | Requires SRC | Native |
| **DEM integration** | Embedded in focusing step | Post-focus resampling | Post-focus resampling |
| **Computational cost (CPU)** | High — $\mathcal{O}(N^3)$ naive | Moderate — $\mathcal{O}(N^2\log N)$ | Moderate |
| **GPU-accelerated** | Yes — near-linear speedup | Partial | Partial |
| **InSAR suitability** | Excellent | Good | Very good |
| **Airborne sensors** | Recommended | Suboptimal | Suboptimal |
| **Spaceborne (linear orbit)** | Excellent with GPU | Very fast, sufficient | Very fast, sufficient |

**Recommendation by use case:**

- **NISAR / Spaceborne stripmap (with GPU):** TDBP preferred for geometric accuracy; CSA acceptable for speed-critical pipelines.
- **UAVSAR / Airborne:** TDBP required — non-linear trajectory makes frequency-domain methods unreliable.
- **Spotlight or circular SAR:** TDBP required — non-rectangular data support incompatible with frequency-domain algorithms.
- **CPU-only, large scene:** CSA or RDA preferred for speed; TDBP with FFBP if phase accuracy is critical.

## 15. Troubleshooting and Common Pitfalls

### Output image is black / all zeros

- Confirm the `orbit` object covers the full data acquisition time span plus at least 30 seconds of margin on each side.
- Verify `fc`, `dt`, `dr` match the actual acquisition parameters — wrong values will cause all geo2rdr iterations to fail.
- Check `side` (LookSide) — swapping left/right causes the algorithm to look in the wrong direction.

### Geometric distortion or misplacement

- Verify the DEM is **WGS84 ellipsoid referenced** (not EGM96/EGM2008 geoid). Use a geoid undulation model (e.g., `geoid-undulation` package) to convert if needed.
- Confirm the DEM EPSG code is correctly set in the GeoTIFF metadata.
- Increase `maxiter` if operating in polar regions or very high squint geometries where Newton's method converges slowly.

### Defocusing / blurry output

- Ensure the input is **range-compressed** data, not raw (unfocused) data. Backprojection performs azimuth compression but not range compression.
- Verify the chirp reference function used for range compression matches the actual chirp parameters (bandwidth $B$, start frequency, FM rate $K_r$).
- Check that the orbit state-vector spacing is $\leq 30$ seconds. Sparse state vectors (> 60 s spacing) can introduce Hermite interpolation errors.

### Phase errors in InSAR products

- Use `flatten=True` to remove the carrier phase offset, centring the interferometric phase on zero.
- Use the sinc or Lanczos kernel (`sincLen=9`) rather than cubic or bilinear for InSAR applications.
- Confirm that both reference and secondary images are focused with **the same DEM** — using different DEMs introduces a topographic phase error.
- If coherence is lower than expected: apply common-band Doppler filtering before backprojection for acquisitions with different Doppler centroids.

### GPU memory exhaustion

- Reduce `blocksize` — this controls how much of the range-compressed data is resident on the GPU at once.
- For very large scenes, process in azimuth blocks and concatenate the output.
- On multi-GPU systems, distribute sub-apertures across GPUs using `isce3.cuda.set_device()`.

### Slow processing on CPU

- Always use `isce3.cuda.focus.backproject` if a CUDA GPU is available — the speedup is typically 50–200×.
- On CPU, enable OpenMP by setting `OMP_NUM_THREADS` to the number of physical cores.
- Consider FFBP (if available in your ISCE3 build) for large scenes processed on CPU.
